# integration spikes

Use this area when a disposable question genuinely requires the real repository,
build, integration boundary, or performance profile.

A spike worktree is not production-bound. Name the question, base commit,
allowed scope, evidence, and deletion condition before editing. Keep the change
small enough that the mechanism remains visible.

Preserve the finding, then reimplement any accepted behavior in a fresh PR
worktree. Do not silently retarget the spike branch into production work.

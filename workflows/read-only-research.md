# Read-Only Research Workflow

Use this workflow when the next useful product is understanding: repository
orientation, unclear bug ownership, unfamiliar runtime paths, branch review, API
discovery, or execution-flow tracing.

The principal prepares one bounded research question. A fresh explorer returns a
compact evidence packet that strengthens the principal's plan without importing
the whole research transcript.

## Assignment Shape

Name:

- the question that must be answered;
- authoritative repositories, branches, and documents;
- allowed read-only tools;
- material unknowns that would change the route;
- evidence and citation expectations;
- the return channel;
- the stopping condition.

For Codex, `repo-explorer` is the default role. Claude uses its configured
GLM-5.2 delegation route with the same read-only assignment boundary.

## Research Stance

Map the smallest complete path that lets the principal act without guessing.
Use targeted search, bounded reads, and compact command output. Trace actual
entry points to behavior. Separate direct observation from inference.

Keep git, files, dependencies, services, and external state unchanged. Runtime
validation belongs in a separately authorized assignment.

## Return Packet

```text
Question answered:
Authoritative state inspected:
Confirmed execution path:
Evidence and locators:
Material unknowns:
Risks:
Recommended next proof-producing action:
```

The principal verifies the returned locators, updates its own plan or control
documents, and dispatches implementation only after the applicable planning
authority is open.
